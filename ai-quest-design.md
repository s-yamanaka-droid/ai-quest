## アーキテクチャ方針

- **現在**：HTML単体。`LESSON_DATA` 配列にSTAGEごとのデータを追加する構造
- **STAGE 5以降実装時**：Next.js + Supabase移行を検討
- **デプロイ**：Vercel（スコープ：s-yamanaka-droids-projects）
- **APIコスト**：ゼロ設計。AI講師はclaude.aiへの誘導（受講生自身のアカウントを使用）

## データ追加ルール

```javascript
// LESSON_DATA配列に以下の形式でオブジェクトを追加する
{
  stage: 1,           // ステージ番号（1〜10）
  lv: 1,              // レベル番号（1〜50）
  num: 'LESSON 01',   // 表示用番号
  title: '...',       // レッスンタイトル
  time: '20 min',     // 目安時間
  diff: '★★☆☆☆',    // 難易度（★1〜5）
  xpVal: 80,          // 取得XP
  blocks: [           // 説明ブロック（2本が標準）
    { icon:'📚', title:'...', time:'...', html:`...` },
    { icon:'⚡', title:'...', time:'...', html:`...` }
  ],
  quiz: {             // 4択クイズ
    q: '質問文',
    opts: ['A','B','C','D'],
    c: 1,             // 正解のインデックス（0〜3）
    ok: '正解時の説明',
    ng: '不正解時の説明'
  },
  ch: {               // ハンズオン課題
    title: '課題タイトル',
    body: '課題の説明文'
  }
}
```
